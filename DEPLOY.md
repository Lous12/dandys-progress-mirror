# Обновление Dandy’s Progress до v11

## 1. Supabase — обязательная миграция

Открой проект Supabase:

1. **SQL Editor** → **New query**.
2. Открой файл `supabase-social-v11.sql`.
3. Скопируй весь текст, вставь в редактор и нажми **Run**.
4. Зелёное сообщение об успешном выполнении означает, что таблицы дружбы и сообщений готовы.

Файл можно запускать повторно. Он создаёт:

- `friendships` — заявки и друзья;
- `user_blocks` — блокировки;
- `direct_messages` — личные сообщения;
- RLS-политики;
- функции проверки дружбы и блокировок;
- подписки Realtime для сообщений и заявок.

## 2. Cloudflare

1. Распакуй архив v11.
2. Замени `supabase-config.js` рабочим файлом из старой версии.
3. Загрузи **все файлы** новым развёртыванием, включая `social.js` и `supabase-social-v11.sql`.
4. SQL-файл будет лежать на сайте, но не исполняется браузером и не содержит секретов.

## 3. GitHub Pages

1. Репозиторий → **Add file** → **Upload files**.
2. Загрузи все новые файлы **кроме `supabase-config.js`**, чтобы не затереть его.
3. Commit message: `Обновление до v11: друзья и сообщения`.
4. Нажми **Commit changes**.
5. Подожди публикацию GitHub Pages и нажми `Ctrl + F5`.

## 4. Проверка

Создай два тестовых аккаунта в разных браузерах или в обычном и приватном окне:

1. На обоих аккаунтах настрой публичный профиль.
2. На первом найди второй аккаунт во вкладке **Друзья → Найти игроков**.
3. Отправь заявку.
4. На втором открой **Заявки** и нажми **Принять**.
5. Отправь сообщения в обе стороны.
6. Проверь счётчик непрочитанных и блокировку.

## Если появляется ошибка таблицы

Значит миграция `supabase-social-v11.sql` не была выполнена или завершилась с ошибкой. Скопируй текст ошибки из SQL Editor — по нему можно точно понять проблемное место.
