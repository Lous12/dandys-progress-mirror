# Бесплатная публикация Dandy’s Progress v5

Схема остаётся полностью бесплатной для небольшого сайта друзей:

- **Cloudflare Pages** — сайт и бесплатный адрес `.pages.dev`;
- **Supabase** — аккаунты, приватные сохранения и публичные страницы игроков.

## 1. Создай проект Supabase

1. Создай бесплатный проект на сайте Supabase.
2. Открой **SQL Editor → New query**.
3. Вставь весь файл `supabase-setup.sql` и нажми **Run**.
4. Даже если ты уже запускал SQL из старой версии, запусти новый файл ещё раз: он добавляет таблицу `public_profiles` и новые политики безопасности.

В базе появятся две таблицы:

- `user_progress` — полное приватное сохранение, доступное только владельцу;
- `public_profiles` — безопасная публичная сводка без email и личных заметок.

## 2. Подключи сайт к Supabase

В **Project Settings → API** скопируй:

- Project URL;
- publishable key или legacy `anon` key.

Вставь их в `supabase-config.js`:

```js
window.SUPABASE_CONFIG = {
  url: 'ТВОЙ_PROJECT_URL',
  anonKey: 'ТВОЙ_PUBLIC_ANON_KEY',
};
```

Никогда не вставляй в сайт секретный `service_role` key.

## 3. Опубликуй на Cloudflare Pages

1. Открой **Workers & Pages**.
2. Выбери **Create application → Pages → Upload assets / Direct Upload**.
3. Загрузи содержимое папки `dandys-tracker`.
4. Cloudflare выдаст адрес вида `dandys-progress.pages.dev`.

Затем в Supabase открой **Authentication → URL Configuration**:

- вставь адрес сайта в **Site URL**;
- добавь его же в **Redirect URLs**.

## 4. Проверь аккаунты и публичные страницы

1. Зарегистрируй первый аккаунт.
2. Подтверди email, если Supabase этого потребует.
3. Нажми на имя аккаунта → **Публичный профиль**.
4. Выбери ник, имя, описание и туна для аватара.
5. Открой раздел **Игроки**.
6. Создай второй аккаунт в другом браузере или режиме профиля и проверь, что страницы видят друг друга.

Ссылка на профиль выглядит примерно так:

```text
https://твой-сайт.pages.dev/#player=ukiol
```

## 5. Что именно видно другим

Публично показываются только:

- отображаемое имя и выбранный ник;
- описание профиля;
- выбранный тун-аватар;
- купленные туны;
- выполненные мастерства;
- проценты исследований и полученные тринкеты;
- общие счётчики вроде максимального этажа и числа машин.

Не публикуются:

- email;
- пароль;
- личные заметки;
- полный приватный JSON сохранения.

Пользователь может выключить переключатель **«Показывать профиль другим игрокам»**. Тогда профиль исчезнет из общего списка, но останется доступен владельцу после входа.

## 6. Перенос старого локального прогресса

Открой новую версию сайта в том же браузере, где был старый прогресс. После первого входа, если облачное сохранение пустое, текущие локальные данные будут отправлены в аккаунт.

На всякий случай перед обновлением сделай **Экспорт JSON**.

## 7. Картинки

Рендеры загружаются с Dandy’s World Wiki/Fandom. Если внешний источник недоступен, сайт показывает инициалы. Для полностью автономной версии позже можно положить изображения в локальную папку `images`.
