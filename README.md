# Dandy’s Progress v7

Фанатский трекер прогресса для Dandy’s World с личными аккаунтами и открытыми страницами игроков.

## Быстрый запуск без аккаунтов

Открой `index.html`. Коллекция, мастерства, исследования и достижения будут сохраняться в браузере.

## Что есть в v7

- Личная страница прогресса.
- Все туны со статусами «Получен», «Мастерство» и «Винтаж», а также твистеды, исследования, тринкеты и достижения.
- Ручные счётчики машин, метров, предметов и особых заданий.
- Более игровой фанатский дизайн в стиле Gardenview.
- Регистрация и вход через Supabase.
- Отдельное приватное облачное сохранение для каждого пользователя.
- Раздел **«Игроки»** с поиском открытых профилей.
- Публичная страница игрока с коллекцией, мастерствами, исследованиями и общей статистикой.
- Выбор туна для аватара, короткое описание и ссылка вида `#player=nickname`.
- Переключатель приватности публичной страницы.
- Локальный гостевой режим, экспорт и импорт JSON.

## Важно

Для аккаунтов и страниц игроков:

1. Заполни `supabase-config.js`.
2. Запусти актуальный файл `supabase-setup.sql` в SQL Editor проекта Supabase.
3. Опубликуй папку сайта на Cloudflare Pages.

Полная инструкция находится в `DEPLOY.md`.

## Изменение v7

Дублирующие отметки «Лицензия» и «Куплен» объединены в единый статус **«Получен»**. Старые сохранения переносятся автоматически; мастерства, винтажные скины и счётчики не сбрасываются.


## Версия v8
Добавлена ненавязчивая вкладка поддержки со ссылкой на Boosty. Изменения базы данных не требуются.

## Русский и английский язык

В версии v9 добавлен переключатель `EN / RU` в верхней панели сайта. Выбранный язык сохраняется в браузере. При первом открытии язык выбирается по языку браузера: русский для русской локали, английский для остальных.

Переведены навигация, страницы, формы, аккаунты, публичные профили, рейтинги, сравнение игроков, задания мастерства, официальные достижения, страница поддержки и политика конфиденциальности.

Файл `i18n.js` обязателен при загрузке сайта.


## Своя аватарка профиля

В редакторе публичного профиля появилось поле **«Своя аватарка (ссылка или путь)»**.

Можно указать:

- полный HTTPS-адрес картинки;
- путь к файлу внутри сайта, например `assets/avatars/ukiol.png`.

Если поле пустое, сайт показывает выбранного туна. Картинка сохраняется вместе с публичным профилем в `public_data`, поэтому новый SQL не требуется.
